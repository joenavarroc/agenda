📅 Documentación API - Agenda de Contactos y Eventos

Proyecto en Java + Spring Boot con autenticación JWT y OAuth. Permite registrar, editar y eliminar contactos y eventos.

🔒 Autenticación

Para acceder a los endpoints protegidos debes enviar un token JWT en el encabezado:

Authorization: Bearer <tu-token>

👤 Usuario

Autenticación básica con usuario y contraseña (registrar/login).

📇 Contactos

➕ POST /contactos

Guardar un nuevo contacto.

Request body:

{
  "nombre": "Juan Pérez",
  "email": "juan@mail.com",
  "telefono": "1122334455"
}

Response 200:

OK

📄 GET /contactos/formulario

Muestra el formulario para agregar un contacto.

❌ GET /contactos/eliminar/{id}

Elimina un contacto de la agenda.

Parámetros:

{id} → ID del contacto

✏️ GET /contactos/editar/{id}

Formulario para editar contacto.

Parámetros:

{id} → ID del contacto

📆 Eventos

🆕 POST /eventos

Crear un nuevo evento.

Request body:

{
  "titulo": "Reunión",
  "descripcion": "Reunión con equipo",
  "fecha": "2025-05-30",
  "hora": "10:00",
  "usuarioId": 1
}

📄 GET /eventos

Listar todos los eventos del usuario.

🔄 PUT /eventos/{id}

Actualizar un evento existente.

Parámetros:

{id} → ID del evento

Request body:

{
  "titulo": "Cumpleaños",
  "descripcion": "Fiesta sorpresa",
  "fecha": "2025-05-26",
  "hora": "14:30",
  "usuarioId": 5
}

🗑️ DELETE /eventos/{id}

Eliminar evento por ID.

{id} → ID del evento

⚙️ Tecnologías Usadas

  • Java 21
  • Spring Boot
  • Spring Security + JWT
  • OAuth 2.0
  • Thymeleaf + HTML + CSS
  • MySQL
  • Swagger para documentación

📌 Notas

  • Usar Swagger en /swagger-ui/index.html para probar la API visualmente.
  • Los endpoints requieren autenticación después del login.
  • Los campos de fecha deben ir en formato yyyy-MM-dd.
